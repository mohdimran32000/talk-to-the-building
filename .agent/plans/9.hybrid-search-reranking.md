# Module 6: Hybrid Search & Reranking

## Context

Currently, retrieval uses **cosine similarity only** (vector search via pgvector). This misses keyword-exact matches — e.g., searching for "UPS" won't rank well if the embedding doesn't capture the acronym semantically. Module 6 adds PostgreSQL full-text search alongside vector search, combines them with Reciprocal Rank Fusion (RRF), and optionally reranks results. Both features are admin-configurable (hybrid ON by default, reranking OFF by default).

**Reranking** supports two providers:
- **Gemini (default)** — LLM-as-judge using `gemini-2.0-flash`, zero new dependencies
- **Cohere (optional)** — dedicated Rerank API, enabled by providing a Cohere API key in admin settings

**Complexity: ⚠️ Medium**

## Task 1: Database Migration

**Create:** `backend/migrations/008_hybrid_search.sql`

1. Add `tsv` column (type `tsvector`) to `document_chunks`
2. Create GIN index on `tsv` for fast full-text search
3. Create a trigger `trg_document_chunks_tsv` that auto-populates `tsv := to_tsvector('english', content)` on INSERT/UPDATE — this means **no Python ingestion changes needed**
4. Backfill existing chunks: `UPDATE document_chunks SET tsv = to_tsvector('english', content) WHERE tsv IS NULL`
5. Create `match_document_chunks_hybrid()` RPC:
   - Parameters: `query_embedding vector(768)`, `query_text TEXT`, `match_user_id UUID`, `match_count INT DEFAULT 20`, `metadata_filter JSONB DEFAULT NULL`, `rrf_k INT DEFAULT 60`
   - CTE `vector_results`: cosine similarity ranked (same as existing), LIMIT `match_count * 2`
   - CTE `keyword_results`: `ts_rank_cd` on `tsv @@ plainto_tsquery('english', query_text)`, LIMIT `match_count * 2`
   - CTE `combined`: FULL OUTER JOIN on chunk `id`, RRF score = `1/(k+vector_rank) + 1/(k+keyword_rank)`
   - Returns `id, document_id, content, rrf_score` ordered by `rrf_score DESC`, LIMIT `match_count`
6. Add columns to `global_settings`:
   - `hybrid_search_enabled BOOLEAN DEFAULT TRUE`
   - `reranking_enabled BOOLEAN DEFAULT FALSE`
   - `reranking_provider TEXT DEFAULT 'gemini'` — `'gemini'` or `'cohere'`
   - `cohere_api_key TEXT` — optional, enables Cohere reranking when set

## Task 3: Hybrid Retrieval in Python

**Modify:** `backend/app/services/openai_client.py` — update `retrieve_chunks()`

```python
def retrieve_chunks(query, user_id, supabase_client, top_k=5, metadata_filter=None):
    from app.services.settings import get_hybrid_search_enabled, get_reranking_enabled
    query_embedding = embed_text(query)
    hybrid = get_hybrid_search_enabled()
    reranking = get_reranking_enabled()

    if hybrid:
        fetch_count = top_k * 4 if reranking else top_k
        result = supabase_client.rpc("match_document_chunks_hybrid", {
            "query_embedding": query_embedding,
            "query_text": query,
            "match_user_id": user_id,
            "match_count": fetch_count,
            "metadata_filter": json.dumps(metadata_filter) if metadata_filter else None,
        }).execute()
    else:
        # Fallback to vector-only (existing RPC)
        result = supabase_client.rpc("match_document_chunks_with_filters", {
            "query_embedding": query_embedding,
            "match_user_id": user_id,
            "match_count": top_k,
            "metadata_filter": json.dumps(metadata_filter) if metadata_filter else None,
        }).execute()

    chunks = [row["content"] for row in (result.data or [])]
    if hybrid and reranking and len(chunks) > top_k:
        chunks = rerank_chunks(query, chunks, top_k)
    return chunks
```

Key: `_execute_search_documents()` delegates to `retrieve_chunks()`, so the tool calling flow (Module 4b) automatically inherits hybrid search — **no changes needed to the tool calling flow**.

## Task 4: Configurable Reranker (Gemini + Cohere)

**Create:** `backend/app/services/reranker.py`

**Main entry point:**
- `rerank_chunks(query: str, chunks: List[str], top_k: int = 5) -> List[str]`
- Reads `get_reranking_provider()` to decide which backend to use
- `@traceable(name="rerank_chunks", run_type="chain")` for LangSmith
- Graceful fallback: if reranking fails, log warning, return first top_k chunks (RRF-ordered)

**Gemini reranker (default):**
- Uses the admin-configured model via `get_llm_model()` (defaults to `gemini-3-flash-preview`) for consistency with the rest of the stack
- Pydantic structured output: `RerankResult` with `rankings: list[ChunkRelevance]` where `ChunkRelevance` has `index: int, score: float`
- Prompt asks LLM to score each chunk 0.0–1.0 relevance to query
- Chunks truncated to 500 chars in prompt to limit token usage
- Sort by score descending, return top_k

**Cohere reranker (optional):**
- Add `cohere` to `backend/requirements.txt`
- Uses `cohere.ClientV2(api_key=get_cohere_api_key()).rerank(model="rerank-v3.5", query=query, documents=chunks, top_n=top_k)`
- Returns reordered chunks based on Cohere's relevance scores
- Only available when `cohere_api_key` is set in admin settings

**Modify:** `backend/app/services/openai_client.py` — add import:
```python
from app.services.reranker import rerank_chunks
```

**Validate:**
- Enable reranking with Gemini provider — verify results and LangSmith span
- Set Cohere API key + switch provider to "cohere" — verify Cohere reranking works
- Disable reranking — retrieval should still work without reranking step
