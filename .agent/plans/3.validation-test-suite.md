# Plan: Comprehensive Validation Test Suite

## Complexity: 🔴 Complex
Multiple test files across backend + frontend. Sequential execution required for Tasks 1-9 (helpers first, then modules). Task 10 (Playwright) is independent. Task 11 validates everything.

---

## Context

The RAG masterclass app has grown through Modules 1, 1b, and 2 with ad-hoc test scripts (`test_module1b_e2e.py`, `test_module2_e2e.py`, `module1.spec.ts`). As new modules are built (3-8), we need a unified test suite an agent can run after any change to verify nothing is broken. This plan creates that suite covering all existing features.

---

## File Structure

```
backend/scripts/
  test_helpers.py       # Shared: auth, SSE parsing, polling, cleanup, test runner
  test_all.py           # Unified runner (imports all suites, aggregates results)
  test_health.py        # Health endpoint
  test_auth.py          # Auth rejection on all endpoints
  test_threads.py       # Thread CRUD
  test_messages.py      # Messages + SSE streaming
  test_files.py         # File upload, ingestion polling, delete
  test_rag.py           # RAG retrieval + memory integration
  test_rls.py           # Row-level security (two-user isolation)

frontend/e2e/
  full-suite.spec.ts    # All UI features (auth, threads, messages, docs, theme)
```

Existing test files (`test_module1b_e2e.py`, `test_module2_e2e.py`, `module1.spec.ts`) remain untouched as historical references.

## Task 1: Create `backend/scripts/test_helpers.py`

Shared module extracted from patterns in existing test scripts.

**Exports:**
- `BASE_URL` — from env, default `http://localhost:8001`
- `SUPABASE_URL`, `SUPABASE_ANON_KEY` — from `.env` via dotenv
- `TEST_USER_A` / `TEST_USER_B` — two test user credential dicts
- `test(name, condition, detail="")` — prints PASS/FAIL, increments counters
- `section(title)` — prints section header
- `get_auth_token(email, password)` — Supabase auth with auto-signup fallback
- `auth_headers(token)` — returns `{"Authorization": f"Bearer {token}", ...}`
- `stream_sse(token, thread_id, content, timeout=60)` — sends message, parses SSE, returns `(full_text, status_code)`
- `poll_document_status(token, doc_id, target="ready", max_wait=30)` — polls GET /api/files every 2s
- `track_thread(thread_id)` — registers a thread ID for scoped cleanup
- `track_file(file_id)` — registers a file ID for scoped cleanup
- `cleanup_threads(token)` — deletes only threads tracked during this test run (NEVER all user data)
- `cleanup_files(token)` — deletes only files tracked during this test run (NEVER all user data)
- `clear_token_cache()` — forces re-authentication on next `get_auth_token()` call
- `summary()` — prints totals, returns exit code

**Reference:** `backend/scripts/test_module2_e2e.py` (lines 1-50 for auth/SSE patterns)

## Task 8: Create `backend/scripts/test_rls.py` (8 tests)

Two-user isolation test.

Setup: auth both USER_A and USER_B. USER_A creates thread + uploads file. Teardown: delete only resources created during this test run (A's thread + file). User B creates no persistent data.

| Test | Criteria |
|------|----------|
| B can't see A's threads | A's thread ID not in B's list |
| B can't get A's thread | 404 |
| B can't delete A's thread | thread still exists for A |
| B can't see A's messages | 404 |
| B can't post to A's thread | 404 |
| B can't see A's files | A's file ID not in B's list |
| B can't delete A's file | 404 |
| A's data intact after B's attempts | both thread and file still in A's lists |

## Task 9: Create `backend/scripts/test_all.py`

Single-process unified runner. Each module exposes a `run()` function returning `(passed, failed)`.

Execution order: health -> auth -> threads -> messages -> files -> rag -> rls

Prints per-suite results and final aggregate summary. Exits with code 0 if all pass, 1 otherwise.

## Task 10: Create `frontend/e2e/full-suite.spec.ts` (26 tests)

Extends existing Playwright patterns. Serial mode. Test user: `test@test.com` / `supabase123`.

**Auth (9 tests):** Protected route redirect, login/signup rendering, navigation links, invalid credentials error, valid login, sign out, session persistence.

**Threads (4 tests):** Create via button, auto-create on message, delete, select (active state).

**Messages (4 tests):** Send shows user bubble, streaming produces response, stop button appears, stop aborts.

**Theme (3 tests):** Toggle visible, switch to dark (html has `dark` class), persists on reload.

**Console (1 test):** No critical errors (filter favicon/extension/404).

**Reference:** `frontend/e2e/module1.spec.ts` for selector patterns, login flow, console error filtering.

## Task 11: Verification

Run the full suite to confirm all tests pass:

```bash
# Backend (from project root)
cd backend && venv/Scripts/python scripts/test_all.py

# Frontend (from project root)
cd frontend && npx playwright test e2e/full-suite.spec.ts
```

**Both servers must be running:**
- Backend: `uvicorn app.main:app --reload --port 8001`
- Frontend: `npm run dev` (port 5173)

**"All tests passing" means:**
- Backend: 63/63 tests pass (health 2 + auth 10 + threads 12 + messages 9 + files 14 + rag 8 + rls 8)
- Frontend: 26/26 Playwright tests pass
- Total: 89 tests covering auth, CRUD, streaming, ingestion, retrieval, memory, RLS isolation, and all UI features
