# Admin Global Settings

**Complexity:** ⚠️ Medium

## Context

All configuration (LLM API key, model name, LangSmith keys) is currently hardcoded or read from env vars. There's no user profiles table and no concept of admin vs regular user. This plan adds a `profiles` table (auto-created on signup with `is_admin` flag), a `global_settings` single-row table, and an admin-only UI to manage LLM and observability settings. Settings are read from DB with env var fallback.

**Scope:** LLM API key + model name, LangSmith API key + project + tracing toggle. Embedding config stays hardcoded (changing it requires re-ingestion — separate concern).
